# GA4 反映ガイド（AI Intern LP v2 計測）

作成日: 2026-05-10
対象プロパティ: `G-M38LFT8GCX`（既にindex.html head内でロード済）

## 1. 送信されるイベント一覧

すべて `event_category=instagram_lp`、`lp_variant`（default / badmove / niche / future）が共通付与される。

| イベント名 | 発火タイミング | 主なパラメータ | 用途 |
|---|---|---|---|
| `lp_view` | DOMContentLoaded時に1回 | utm_source / utm_medium / utm_campaign / utm_content / utm_term / event_label=variant | 流入ボリューム・流入元別の母数把握 |
| `section_view` | セクションが45%可視で初回1回 | section_id（hero / problem / experience / evidence / skills / principles / flow / fit / requirements / application_flow / faq / apply_form / final_cta / related_links） | 読了ファネル・離脱位置 |
| `form_reached` | `.form-frame` が30%可視で1回 | event_label=form_reached | フォーム到達（中間CV） |
| `form_inview_full` | `.form-frame` が80%可視で1回 | intersection | フォームが完全に見えた率 |
| `form_focus` | iframe（Airtable）にフォーカス | — | 強い記入意思のシグナル |
| `cta_click` | `.js-track[data-event="cta_click"]` クリック | cta_position（hero_form / skills_form / fit_form / final_form / sticky_form / form_direct / header_form） | 各位置の押下量 |
| `dm_click` | `.js-track[data-event="dm_click"]` クリック | cta_position（hero_instagram / skills_instagram / fit_instagram / final_instagram / sticky_instagram / form_instagram / header_instagram） | DM相談の発生量 |
| `outbound_click` | 外部ドメイン（jsトラック対象外）リンクのクリック | link_url / link_domain | Podcast・コンテストLP・親サイトへの離脱経路 |
| `scroll_depth` | 25 / 50 / 75 / 90% 到達時 | scroll_depth | 読了率 |
| `engaged_time` | 可視+アクティブ時間が15 / 30 / 60 / 120 / 300秒到達時 | engagement_seconds / engagement_time_msec | 滞在の質 |
| `lp_exit` | beforeunload時 | final_engagement_seconds / max_scroll_depth | 離脱時のサマリ |

クリックイベントは600ms以内の重複を抑止（誤タップ・ダブルクリック対策）。

## 2. GA4 管理画面でのセットアップ（推奨）

### 2-1. カスタム定義（イベントスコープ）に登録

「管理 → データ表示 → カスタム定義 → カスタムディメンションを作成」で以下を**イベントパラメータ**として登録すると、探索レポートで使えるようになる。

| ディメンション名 | パラメータ名 | スコープ | 用途 |
|---|---|---|---|
| LP Variant | `lp_variant` | イベント | A/B/C案別のCVR比較 |
| Section ID | `section_id` | イベント | どこまで読まれているか |
| CTA Position | `cta_position` | イベント | hero / fit / final など位置別CTR |
| Outbound Domain | `link_domain` | イベント | 外部離脱の内訳 |
| Engagement Seconds | `engagement_seconds` | イベント | 滞在質の階段把握 |
| Scroll Depth | `scroll_depth` | イベント | 読了率 |
| Final Engagement | `final_engagement_seconds` | イベント | 離脱時の滞在 |
| Max Scroll | `max_scroll_depth` | イベント | 離脱時の到達 |

### 2-2. ユーザープロパティ

`gtag('set','user_properties',{...})` で送られるので、**ユーザースコープ**で登録：

| プロパティ表示名 | パラメータ名 | 用途 |
|---|---|---|
| LP Variant (User) | `lp_variant` | 同一ユーザーの再訪をvariantで属性化 |
| LP UTM Source | `lp_utm_source` | 再訪元のラベル化 |
| LP UTM Campaign | `lp_utm_campaign` | キャンペーン保持 |

### 2-3. キーイベント（旧コンバージョン）に指定

「管理 → イベント → キーイベントとしてマーク」で以下をONにする：

- `cta_click`（応募フォーム到達意思）
- `dm_click`（DM相談意思）
- `form_focus`（記入意思）
- `form_reached`（フォーム到達）

応募フォーム送信そのものはAirtable iframe内のため直接捕捉不可。フォーム送信完了後にAirtable側でリダイレクトURLを設定し、そこに `?submitted=1` を付け、index.html側で検知して別途 `application_submit` イベントを発火する案も検討可能（必要なら実装します）。

## 3. UTM運用ルール

Instagram投稿ごとにURLを分ける。LPは共通で受け、`src` または `utm_content` がHeroコピーを切り替える。

```
https://ai-intern.metagri-labo.com/?utm_source=instagram&utm_medium=feed&utm_campaign=intern_202605&utm_content=badmove
                                                                                              &utm_content=niche
                                                                                              &utm_content=future
```

DM個別送付・ストーリーズリンクの場合は `utm_medium` を `dm` / `story` に変える。

## 4. DebugView での動作確認手順

1. Chromeに「Google Analytics Debugger」拡張を入れる、もしくはURL末尾に `?gtm_debug=x` を付けて開く
2. GA4管理画面 → 設定 → DebugView を開く
3. LPで以下を順に実行
   - ページ閲覧 → `lp_view` が出る
   - 各セクションをスクロール → `section_view` が13回（hero〜related_links）順に出る
   - スクロール深度を上げる → `scroll_depth` が25/50/75/90で出る
   - 60秒以上タブを開く → `engaged_time` が15/30/60で順に出る
   - ヘッダー「DMで相談」をクリック → `dm_click`（label=header_instagram）
   - フッター「Podcastを確認する」をクリック → `outbound_click`（domain=podcasts.apple.com）
   - フォームエリアまでスクロール → `form_reached` → `form_inview_full`
4. `lp_variant` が default / badmove / niche / future のいずれかでイベントに付いていることを確認

## 5. 探索レポートのテンプレ案

| レポート名 | ディメンション | 指標 | 見るもの |
|---|---|---|---|
| Variant別CVRファネル | lp_variant × イベント名 | イベント数・ユーザー数 | A/B/C案のCTAクリック率比較 |

| 流入元 × 応募意欲 | utm_source / utm_content × cta_click | イベント数 | 投稿テーマ別の効果 |

## 6. 既知の制約

- **Airtableフォーム送信**は同一オリジンでないため、JSから完了検知不可。必要なら送信後リダイレクト先で計測ページ（thanks.html）を設置する。
- **iframeフォーカス検知**（`form_focus`）はブラウザによって挙動が異なる。Safariでは取りこぼす可能性あり。Airtable到達は `form_inview_full` で代替指標として参照可能。
- **背景タブ**では `engaged_time` が止まる仕様（visibleかつ直近5秒以内のアクティビティが必要）。ユーザーがタブを離れた時間は加算されないため、過大計測にならない。
- **prefers-reduced-motion**ユーザーには動的演出が無効化されるが、計測は通常通り動く。
