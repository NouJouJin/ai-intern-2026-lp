# Instagram流入専用LP 要件整理

作成日: 2026-05-09

## 結論

Instagramから流入する専用LPは、「インターン募集の説明ページ」ではなく、「AI時代にどんな経験を積むべきかを問い直し、Metagri研究所のインターンに進む理由を作るページ」として設計する。

LPの役割は、以下の3つ。

1. Instagram投稿で生まれた違和感や興味を、応募意欲に変える
2. Metagri研究所のインターンが普通の募集と何が違うかを伝える
3. DM、説明会、応募フォームのいずれかに迷わず進ませる

## LPの基本コンセプト

### 中心メッセージ

```text
AIに聞いて終わらない。
農業の現場で、問いを立て、発信し、改善するインターン。
```

### 伝えるべき価値

「AIを学べる」だけでは弱い。AI時代に価値が残るのは、AIを使って現場にぶつけ、反応を見て、改善し、成果物として残す経験である。

Metagri研究所のインターンは、以下を横断して経験できる点を打ち出す。

- 農業現場の課題理解
- AI活用と情報整理
- Instagram、note、レポートなどの発信
- Discordを中心としたコミュニティ運営
- 農家、参加者、チームを巻き込む企画運営
- 日誌、完了ログ、振り返りによる成長の可視化

## 想定流入

### 流入元

- Instagramフィード投稿
- Instagramリール
- Instagramストーリーズ
- Instagramプロフィールリンク
- DMで送る個別案内リンク

### 投稿別の入口

| 入口 | 投稿テーマ | LPで受け止める言葉 |
|---|---|---|
| A案 | 悪手チェック型 | 「当てはまった人へ。AI時代に残る経験を積もう」 |
| B案 | ニッチ人材募集型 | 「普通のインターンでは物足りない人へ」 |
| C案 | 3年後成長機会型 | 「3年後に効く経験を、今から積む」 |

LP本体は共通でよいが、可能であればURLパラメータや複製ページでファーストビューの見出しだけ出し分ける。

```text
/intern?src=badmove
/intern?src=niche
/intern?src=future
```

## メインターゲット

### 主対象

- 大学生、高専生、専門学生
- AIを使っているが、実践経験に不安がある人
- 農業、地方、食、コミュニティ、発信に関心がある人
- 普通のガクチカや肩書きづくりに違和感がある人
- 将来、企画、マーケティング、編集、事業開発、地域領域に関わりたい人

### 合いやすい人

- 未完成でも一度出して、直せる人
- 指示待ちより、目的を確認して動ける人
- AIの回答を鵜呑みにせず、現場に照らして考えられる人
- 日誌やレポートで学びを残せる人
- DiscordやSNS上で早めに相談できる人
- 農家やコミュニティの声を聞くことに抵抗がない人

### 合いにくい人

- AIツールの使い方だけを学びたい人
- 完成するまで誰にも見せたくない人
- 指示された作業だけをこなしたい人
- 肩書きや実績の見え方だけを重視する人
- 日誌、報告、振り返りを面倒に感じる人

## 主要CTA

### 第一CTA

```text
インターン詳細を聞く
```

遷移先候補:

- 応募フォーム
- 説明会申込フォーム
- Googleフォーム
- Notionフォーム

### 第二CTA

```text
DMで相談する
```

Instagram流入では、いきなり応募よりもDM相談の方が心理的ハードルが低い。LP内に「迷っている人はDMでOK」という逃げ道を用意する。

### CTA配置

- ファーストビュー直下
- 「得られる経験」セクション後
- 「合う人・合わない人」セクション後
- FAQ後
- ページ最下部

スマホでは下部固定CTAも検討する。

## ページ構成

### 1. ファーストビュー

目的: 3秒で「普通の募集ではない」と伝える。

必要要素:

- 強い見出し
- 1文の価値説明
- 第一CTA
- 第二CTA
- 「農業 × AI × 発信 × コミュニティ」の明示

見出し案:

```text
AIに聞いて終わらない。
農業の現場で、問いを立てるインターン。
```

補足文案:

```text
Metagri研究所では、農家の声、AI活用、SNS発信、コミュニティ運営を横断して、3年後にも残る実践経験を積むインターンを募集しています。
```

### 2. 問題提起

目的: Instagram投稿で刺した違和感をLP内でも再確認する。

見出し案:

```text
AI時代に、ただの作業経験は価値が落ちていく。
```

内容:

- AIで文章や資料は作れるようになった
- だからこそ、現場で問いを立てる力が重要になる
- 「誰のために、何を改善するのか」を考える経験が差になる

### 3. Metagriインターンでできること

目的: 具体的な経験を見せる。

掲載項目:

- 農家インタビューや現場課題の整理
- AIを使った調査、要約、企画案づくり
- Instagram、note、レポート制作
- Discordコミュニティの運営補助
- イベントや企画の準備、実行、振り返り
- 学びを日誌、完了ログ、評価に残す

見せ方:

- 6つのカード形式
- 各カードは「やること」「身につく力」をセットで書く

### 4. 得られる力

目的: インターン経験が将来どう効くかを伝える。

訴求する力:

- 目的を確認する力
- 6割で出して改善する力
- AIを使っても現場感を失わない力
- チームに早く相談する力
- 企画を発信物に落とし込む力
- フィードバックを受けて行動を変える力
- 自分の学びを言語化して残す力

### 5. Metagri 5原則

目的: どんな行動が評価されるかを明確にする。

掲載する原則:

1. なぜを追及
2. 6割で進む
3. 即レスSOS
4. DAOオーナー意識
5. フィードバック受容

LPでは長く説明しすぎず、各原則を1文で見せる。

例:

```text
6割で進む: 完璧を待たず、早く出してフィードバックで直す。
```

### 6. 活動の流れ

目的: 応募前の不安を減らす。

掲載項目:

- 週次の活動イメージ
- 日誌、タスク、フィードバックの流れ
- Discordでの相談
- 定期的な振り返り
- クイズや評価による学習確認

例:

```text
調査する → 6割で共有する → フィードバックを受ける → 修正する → 完了ログに残す
```

### 7. 合う人・合わない人

目的: 応募者の自己選抜を促す。

合う人:

- 現場にぶつけながら学びたい
- AIを使って企画や発信に挑戦したい
- 未完成でも出して直せる
- 農業や地域の課題に関心がある
- 学びをログとして残したい

合わない人:

- 受け身で指示だけ待ちたい
- 完璧になるまで出したくない
- AIツールの操作だけを学びたい
- 報告や振り返りを避けたい
- すぐに派手な実績だけが欲しい

### 8. 募集要項

目的: 応募判断に必要な条件を明確にする。

最低限必要な項目:

- 対象者
- 活動期間
- 活動頻度
- 活動場所、オンライン可否
- 主な業務内容
- 使用ツール
- 選考フロー
- 報酬、交通費、特典の有無
- 募集人数
- 締切

未確定項目は「調整中」とせず、LP公開前に決める。

### 9. 応募フロー

目的: 次に何をすればよいかを迷わせない。

例:

```text
1. フォーム送信
2. DMまたはメールで連絡
3. オンライン説明
4. トライアル課題または面談
5. 活動開始
```

### 10. FAQ

目的: 応募前の離脱理由を減らす。

入れるべき質問:

- 農業経験がなくても参加できますか？
- AI初心者でも大丈夫ですか？
- どれくらい時間が必要ですか？
- 地方在住でも参加できますか？
- どんな成果物を作りますか？
- 学校やアルバイトと両立できますか？
- 選考では何を見ますか？
- まず話を聞くだけでも大丈夫ですか？

### 11. 最終CTA

目的: 読了後に迷わせず行動させる。

見出し案:

```text
AI時代に、現場で学ぶ経験を積みたい人へ。
```

CTA:

```text
インターン詳細を聞く
DMで相談する
```

## デザイン要件

### 基本方針

- スマホ閲覧を最優先
- Instagramアプリ内ブラウザで読みやすくする
- 1セクション1メッセージに絞る
- 文字を詰め込みすぎない
- 汎用的な採用サイト感を避ける

### トーン

- 尖っているが、怖すぎない
- 知的だが、堅すぎない
- 学生向けだが、軽すぎない
- 農業らしさとAIらしさを両方出す

### 推奨ビジュアル

- 農業現場の写真
- DiscordやNotionなどの活動画面を抽象化した図
- 投稿、日誌、レポートなど成果物のモック
- 2026年から3年後への成長タイムライン
- チェックリスト、ステップ図、比較表

### 避けたいビジュアル

- 汎用的なスーツ学生写真
- AIっぽいだけの近未来画像
- かわいいだけの装飾
- 意味のないグラデーション背景
- 条件説明だけの無機質なページ

## 計測要件

### UTM設計

Instagram投稿ごとにURLを分けて計測する。

例:

```text
/intern?utm_source=instagram&utm_medium=feed&utm_campaign=intern_202605&utm_content=badmove
/intern?utm_source=instagram&utm_medium=feed&utm_campaign=intern_202605&utm_content=niche
/intern?utm_source=instagram&utm_medium=feed&utm_campaign=intern_202605&utm_content=future
```

### 計測イベント

最低限、以下を計測する。

| イベント | 意味 |
|---|---|
| LP閲覧 | Instagramからどれだけ流入したか |
| CTAクリック | 応募・説明会に進む意欲があるか |
| DMクリック | 軽い相談ニーズがあるか |
| フォーム到達 | 応募前の本気度があるか |
| フォーム送信 | 実応募につながったか |
| スクロール率 | どこまで読まれているか |

### 見るべきKPI

- 投稿ごとのLPクリック率
- LP内CTAクリック率
- フォーム送信率
- DM発生数
- 応募者の質
- A/B/C別の応募率

いいね数だけで判断しない。Instagram投稿の反応とLP内行動をセットで見る。

## フォーム要件

### 必須項目

- 名前
- 連絡先
- 学校名または所属
- 学年
- Instagramアカウント
- 興味を持った理由
- やってみたいこと
- 週に使える時間

### 任意項目

- AIツールの利用経験
- SNS、note、動画、デザインなどの経験
- 農業、地域、食への関心
- ポートフォリオや過去の成果物

### 重要な設問

応募者の質を見るため、以下のような自由記述を入れる。

```text
AI時代に、自分が身につけたいと思う力は何ですか？
```

```text
Metagri研究所のインターンで、どんな経験をしてみたいですか？
```

## 運用要件

### 公開前に決めること

- 募集人数
- 活動期間
- 活動頻度
- 報酬や特典の有無
- 応募締切
- 説明会の有無
- 応募後の返信担当
- DM対応テンプレート
- 不採用または保留時の返信方針

### 公開後に見ること

- どのInstagram投稿から流入したか
- どのCTAが押されたか
- どこで離脱したか
- DM内容にどんな質問が多いか
- 応募者の自己紹介が濃いか

### 改善サイクル

```text
投稿する
→ LP流入を見る
→ DMとフォーム内容を見る
→ FAQやコピーを修正する
→ 次の投稿に反映する
```

## MVP要件

最初に作るLPは、作り込みすぎない。以下があれば公開できる。

- ファーストビュー
- Metagriインターンでできること
- 得られる経験
- 合う人・合わない人
- 募集要項
- 応募フロー
- FAQ
- 応募フォームまたは説明会フォーム
- DM導線
- UTM計測

## Phase 2で追加するもの

初回公開後、反応を見ながら追加する。

- A/B/C別のファーストビュー出し分け
- インターン生の声
- 過去の成果物例
- 活動風景の写真
- 1分紹介動画
- 説明会アーカイブ
- 応募前診断コンテンツ
- よくある質問の追加

## LP公開前チェックリスト

- スマホで見たときに3秒で内容が伝わる
- CTAがファーストビュー内にある
- 募集条件が明確である
- DM導線がある
- 応募フォームが短すぎず長すぎない
- A/B/C投稿ごとのUTMが分かれている
- FAQで不安を減らしている
- 「AIを学べる」だけの訴求になっていない
- 「農業 × AI × 発信 × コミュニティ」が伝わる
- 応募後の返信フローが決まっている

## 推奨する初期構成

最初は1ページ構成でよい。

```text
ファーストビュー
問題提起
できること
得られる力
5原則
活動の流れ
合う人・合わない人
募集要項
応募フロー
FAQ
最終CTA
```

Instagram投稿はA/B/Cで検証し、LPは共通の受け皿にする。反応が最も良い投稿テーマが見えた段階で、そのテーマに合わせた専用ファーストビューを追加するのがよい。
